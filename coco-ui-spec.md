# Coco UI Specification

## Design Philosophy

**"Quiet until needed."**

Coco's interface stays out of your way. No visual noise, no competing elements, no gratuitous decoration. Every pixel earns its place. The UI reveals depth as you navigate deeperâ€”simple at first glance, powerful when you dig in.

## Responsive Behavior

### Breakpoints

```css
--coco-breakpoint-sm: 640px;
--coco-breakpoint-md: 768px;
--coco-breakpoint-lg: 1024px;
--coco-breakpoint-xl: 1280px;
```

### Adaptations

**< 768px (Mobile/Tablet):**
- Wallet cards stack vertically
- Workspace grid becomes single column
- Drawer becomes full-width overlay
- Transaction sidebar collapses to bottom sheet

**768px - 1024px (Small Desktop):**
- Wallet cards: 3 visible, scroll for more
- Workspace grid: 2 columns
- Drawer: 60% width

**> 1024px (Desktop):**
- Full layout as designed
- Wallet cards: 4+ visible
- Workspace grid: 3-4 columns
- Drawer: 50% width

### Transitions

- **Progress**: Horizontal bar, indeterminate or determinate

## Accessibility

### Focus States

All interactive elements have visible focus rings:
```css
outline: 2px solid var(--coco-accent);
outline-offset: 2px;
```

### Color Contrast

- All text meets WCAG AA standards (4.5:1 for normal text, 3:1 for large text)
- Status colors have sufficient contrast in both light and dark modes
- Never rely on color aloneâ€”always pair with icons or text

### Screen Reader Support

- Semantic HTML structure
- ARIA labels for icon-only buttons
- Live regions for status updates (build complete, transaction sent)
- Proper heading hierarchy

## Summary

Coco's UI is built around a clear hierarchy: chains contain wallets and workspaces, workspaces contain contracts and transactions, transactions contain runs. Navigation flows naturally through these layers. The interface stays minimalâ€”white space, subtle borders, restrained colorâ€”until you need depth, then it reveals itself through drawers, modals, and expandable sections. Dark mode inverts the palette without changing the structure. Everything is keyboard-accessible, everything animates smoothly, everything gets out of your way until you need it.
